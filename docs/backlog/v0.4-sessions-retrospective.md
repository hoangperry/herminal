## 2. What we learned

### The foundation was the unlock, not the features

All three slices depend on one thing: **a pane knowing its own cwd**.
That was an unhandled libghostty action (`GHOSTTY_ACTION_PWD`) found
during the v0.2.x stub audit. Wiring OSC 7 in v0.4.0 made the Claude
browser's "resume in the right dir", session restore, and named
workspaces all fall out cheaply. Spending the first half-day on the
foundation paid for the next two slices.

### Read the tool's own data instead of reinventing it

The Claude session browser doesn't track Claude sessions itself — it
reads Claude Code's own transcript store. Zero bookkeeping, always in
sync, survives herminal reinstalls. The lesson: when integrating with
a tool that already persists state, read its store rather than
shadowing it.

### The slug is lossy — parse the body

`~/.claude/projects/<slug>` encodes cwd as `/`→`-`, which is
ambiguous (`andromeda-next` vs `andromeda/next`). The fix — read the
real `cwd` from the transcript body, never decode the slug — is the
kind of correctness detail that only shows up when you test against
real data (40 projects, several with hyphens). Verified before ship.

### "Restore layout, not commands" is the safe default

Session restore + named workspaces both replay the layout + cwd as
plain shells, never re-running ssh/claude. Auto-reconnecting SSH or
resuming an LLM on every launch would be surprising and
side-effectful. Validating each cwd against the local filesystem
(stale/remote → home) makes a former ssh pane degrade gracefully
instead of failing to spawn. Conservative-by-default, with the
explicit actions (the Claude browser's Resume, the SSH panel's
Connect) still one click away.

### A persistence gate avoids clobbering the snapshot

`sessionPersistenceEnabled` starts false so the init/restore tab
churn doesn't overwrite the saved session before AppDelegate has
decided whether to restore. Small flag, prevents a real
order-of-operations bug.

## 3. What didn't ship (deferred, with reasons)

- **Re-running commands on restore.** Deliberately out — see above.
  A future per-pane "remember this command" opt-in could layer on.
- **Live cwd in the status bar / tab title.** OSC 7 is tracked but
  only consumed by restore + the browser. Surfacing it in the UI is
  a nice-to-have, not requested.
- **Named-workspaces sidebar.** Shipped via the Window menu +
  palette instead of a 5th sidebar — lower surface area, same reach.
  A panel could come if the menu proves clunky at 10+ workspaces.

---

## 4. Carry-forward

- **No tech debt opened.** Each slice is self-contained; the stores
  are independent files.
- **No programmable regression-guard for restore yet.** The
  snapshot/restore round-trip was verified by hand (diary log +
  workspace.json inspection). A `verify-session-restore.sh` that
  launches with a crafted snapshot and asserts `dumpState` would
  close the same gap the clipboard/title guards close. Candidate for
  the next dogfood-hardening pass.
- **Site is current** (v0.4.2, Sessions section shows all three).
